# 11.3. Challenge Detail (sub)

**Purpose:** The single page for a public goal `Challenge` — full info, your progress against the target, a join/leave action, and the complete leaderboard (not just the top-3 preview shown on the card). Reached by tapping any Challenge card or row from the #11 Social Challenges tab.

Handles every `Challenge` regardless of `visibility` (public / invite_only) or `metricKind` (accumulator / best_of). Internal layout branches: accumulator challenges show the **Your progress** bar; best_of challenges skip it (no target — the leaderboard below carries the standings signal). Ended best_of challenges also have a wrapping `challenge_result` Post in the Community feed (auto-created on deadline), which links to #11.1 Post Detail for the comment thread.

## Layout

Vertical, scrollable. **No bottom nav** (sub-page). Back arrow → Social (#11).

## UI elements

### Top bar
← back arrow + `CHALLENGE` label in caption-bold.

### Hero
- Top row: `{icon} {shortName}` in accent caps + `· Day X / Y` muted (or `· Ended` after the deadline) + right-aligned `N joined`
- Title in Display 26 px font-black uppercase
- Description below in 14 px ink (full paragraph, not clamped — this is the place to read the full tagline)

### Your progress (joined + accumulator only)
A larger version of the card's progress bar — 3 px tall, current / target row + percentage in 18 px accent font-black on the right. Hidden entirely when the user hasn't joined the challenge yet, OR when `metricKind = 'best_of'` (no target to fill toward — the leaderboard below carries the standings signal).

### Join / Leave action (pinned footer)
**Pinned to the bottom of the screen** in its own footer band — stays reachable regardless of scroll position. A subtle `border-t border-faint/40` separates it from the scrollable content above.

Full-width 14 px font-black uppercase pill:
- `Join Challenge` → filled `accent` CTA when not joined
- `Leave Challenge` → outlined `bg` chip with `ring-1 ring-faint`, muted text, `danger` hover
- For **ended challenges** the button is replaced by a `surface` caption — *"This challenge has ended."* — since joining is no longer possible.

Backed by `joinChallenge(challengeId, userId)` / `leaveChallenge(challengeId, userId)` actions. Label flips on tap; same instant-toggle pattern as the friend toggle on #11.2.

**Why pinned and not inline.** The leaderboard can be long enough to push an inline button off-screen, especially on Flutter's iPhone-sized viewport. Pinning the CTA at the bottom matches Strava / typical "detail-page with primary action" conventions and means users don't have to scroll back up to commit.

### How it works
Auto-derived copy from the challenge's `metric` + `workoutTypeId` + window. Examples:
- accumulator + `total_distance` + running: *"Adds up your running distance between May 1–31, 2026. The highest total wins the leaderboard."*
- accumulator + `active_days` + any workout: *"Counts distinct days where you completed any workout between May 1–31, 2026. The highest count wins the leaderboard."*
- best_of + `fastest_time` + running: *"Each participant's fastest qualifying running session between May 24–25, 2026 is ranked head-to-head. Lowest time wins."*
- best_of + `longest_distance` + any workout: *"Each participant's longest qualifying any-workout session between May 20–31, 2026 is ranked head-to-head. Highest distance wins."*

Stays accurate when the seed changes — no manual upkeep.

### Leaderboard
The **full** list of ranked participants (no top-3 truncation, no `…` separator). Each row:
- Medal (🥇 🥈 🥉 for ranks 1–3) or `N.` for rank ≥ 4
- Name (your row says `You` and gets the accent highlight + `bg-accent/10` + accent ring)
- Metric value right-aligned (e.g. `54.2 km`, `5`)

Zero-progress participants are excluded (matches card preview behavior). When the list is empty: muted prompt *"No one has made progress yet. Be first on the board."*

**Other-user names link to their #11.2 profile.** Your own row is non-clickable (no self-profile link needed from here — you're already in your own context). Consistent with how challenge-result ranking rows link on #11 / #11.1.

## Edges

- **From:**
  - Social (#11) — tap any Challenge card on the Joined sub-tab, OR tap the icon/name area of an Active / Past Challenge row
- **To:**
  - Back (←) → **browser-history back** via `navigate(-1)` (preserves the Social sub-tab state the user came from — Joined / Active / Past — instead of always dumping them on the default Joined tab)
  - User Profile (#11.2) — tap any non-self leaderboard row's name

## Data touched

See [../../database-v1.md](../../database-v1.md).

- **Reads:** the specific `Challenge` (by `ChallengeID` query param); all `ChallengeParticipant` rows for it (for participant count + joined check + per-row ranking); the current user's `WorkoutSession` rows in the window (for the user's progress bar); every participant's `WorkoutSession` rows in the window (for the full leaderboard); `WorkoutType` for the auto-derived "How it works" copy
- **Writes:**
  - `joinChallenge(challengeId, currentUserId)` — Join Challenge tap (no-op if already joined; idempotent)
  - `leaveChallenge(challengeId, currentUserId)` — Leave Challenge tap

## Notes / non-obvious

- **One detail page, every Challenge kind.** Public + invite_only, accumulator + best_of — all four combos route here. The progress bar / leaderboard branches keep the visual right per kind without duplicating the page.
- **Progress + leaderboard are computed live.** Same `computeChallengeProgress` + `rankChallengeParticipants` helpers as the Social card preview, now in `lib/challengeProgress.ts` so both surfaces stay in lockstep with no stored values to drift.
- **Self row on leaderboard is non-tappable.** Linking to your own profile from your own leaderboard row would loop. Other rows link to #11.2 normally.
- **Ended challenges show a caption, not the button.** Cleaner than disabling the Join button — clarifies *why* there's no action and avoids the "is this broken?" question users sometimes have with disabled CTAs.
- **No share button here in v1.** The post-style Share modal lives on #11.1 Post Detail (workout shares + challenge-result posts). If we want shareable challenge progress, add it here later — same `SharePostModal` could be retargeted to share the challenge URL.
- **Numbering:** decimal sub of #11 Social, following the `11.x` pattern of single-parent sub-pages. Sibling of #11.1 Post Detail and #11.2 User Profile.
- **Premium (future):** challenge-specific badges on completion (needs Badge + UserBadge entities), live progress notifications, "your closest competitor is X ahead" hints.
