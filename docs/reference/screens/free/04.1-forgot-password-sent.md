# 4.1. Forgot password — link sent (sub-state of #4)

**Purpose:** The state #4 enters immediately after the user taps "Send reset link". Renders #4's full layout with a confirmation `Modal` overlay on top.

This is the **first sub-state pattern** in the inventory. Convention: `NN.M-<slug>` files describe a discrete UI state of screen `NN` that's worth seeing in isolation in the explorer. Build them as a wrapper component that passes the relevant prop into the parent screen (here: `<ForgotPassword sent />`).

## What's added on top of #4

A centred `Modal` overlay inside the phone's app content area:

- *Backdrop:* `bg/70` with `backdrop-blur-sm`. Tapping it dismisses (when wired) — currently static for the mock.
- *Card:* `surface`, rounded-3xl, 28 px padding, 1 px `faint` ring, drop shadow. Max width ~85% of the phone.
- *Card content (top → bottom, centred):*
  - 56 × 56 px accent circle with a 28 × 28 px `bg`-coloured checkmark inside
  - **"LINK SENT."** title — `text-3xl` (30 px display, font-black, uppercase, tracking-tight). Note the trailing period — same brand pattern as "FORGOT PASSWORD."
  - Body in **Subheadline** (15 px `muted`, centred, `leading-relaxed`): *"Check your inbox at **mia.patel@email.com**. The link expires in 30 minutes."* The email itself is `ink` to highlight what was sent and where.
  - Primary CTA "GOT IT" — full-width `accent` button, **Body** (17 px) font-black uppercase tracking-wider
  - Secondary text link "Didn't get it? Resend in 60s" — **Footnote** (13 px `muted` font-semibold)

The underlying form (#4) stays present behind the backdrop so dismissing the modal returns to a familiar state.

Colours from [../../palette.md](../../palette.md), type sizes from [../../typography.md](../../typography.md). Reuses: `Modal` (new, `app/src/components/Modal.tsx`).

## Edges

- **From:** #4 Forgot password (tap "Send reset link")
- **To:**
  - → #4 Forgot password (tap "Got it" or the backdrop — dismisses the modal)
  - → resend action (tap "Resend in 60s" once cooldown elapses) — stays on this state; server issues another `PasswordResetToken`

## Data touched

Same as #4 — see [../../database-v1.md](../../database-v1.md). This sub-state doesn't add reads or writes; it's a visual confirmation of the action taken on #4.
