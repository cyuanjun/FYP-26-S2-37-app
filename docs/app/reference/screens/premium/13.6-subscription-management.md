# 13.6. Subscription Management (sub) — Premium

**Purpose:** Premium-only screen for viewing and managing the active subscription — plan, price, renewal date, payment method, billing history, and cancel / resume. Reached from a **Premium-only** "Manage Subscription" row on #13 Profile (the row is hidden for Free users). This is the counterpart to #16 Upgrade: #16 *sells* Premium, #13.6 *manages* it.

## Layout

Vertical, scrollable. **No bottom nav** (sub-page). Back arrow → Profile (#13).

1. **Top bar** — ← back + `SUBSCRIPTION` title (caption-bold uppercase).
2. **Plan card** — tier, status pill, price, renewal / access-until line.
3. **Payment method** — mock card label.
4. **Billing history** — list of past charges.
5. **Manage action** — Cancel (when active) / Resume (when cancelled).

## UI elements

### Plan card
Accent-ringed `surface` card: ⭐ + `PREMIUM MONTHLY`, a **status pill** (Active / Cancelled / Past due), the price (`$9.99 / mo`), and a divider line below showing either *"Renews {date}"* (active) or *"Cancelled — access until {date}"* (cancelled).

### Payment method
Single `surface` row with a card glyph + `Subscription.paymentMethodLabel` (mock, e.g. "Visa •••• 4242").

### Billing history
List of past monthly charges. **Mock-derived** — synthesised as one charge per month from `Subscription.startedAt` up to `TODAY` (most-recent first, capped at 12). A real backend reads `BillingRecord` rows; the mock generates them so the list isn't empty. Each row: label + date (left), amount (right).

### Manage action (bottom)
- **Active** → `Cancel subscription` (danger outline). Opens a confirm modal: *"You'll keep Premium access until {renewsAt}. After that you'll return to Free."* Confirming calls `cancelSubscription` (status → `cancelled`, sets `cancelledAt`).
- **Cancelled** → `Resume subscription` (accent fill). Calls `resubscribe` (status → `active`, clears `cancelledAt`).

## Edges

- **From:** Profile (#13) — Premium-only "Manage Subscription" menu row
- **To:**
  - Back (←) → Profile (#13)
  - Go Premium → #16 Upgrade (only in the defensive no-subscription state)

## Data touched

See [../../database-v1.md](../../database-v1.md).

- **Reads:** the current user's `Subscription` row (tier, status, renewsAt, priceCents, paymentMethodLabel, startedAt for billing history).
- **Writes:** `cancelSubscription` / `resubscribe` mutate `Subscription.status` + `cancelledAt`.

## Notes / non-obvious

- **Premium-only entry.** The Profile menu row is gated on `user.role !== 'free'`. A Free user has no `Subscription` row anyway; if the screen is reached directly while Free it shows a defensive "No active subscription · Go Premium" state rather than crashing.
- **Cancel keeps `role = 'premium'` in the mock.** Real billing keeps access until period end then downgrades via webhook. The mock leaves the role intact so the cancelled state is still viewable; a production scheduler would flip `role` to `free` at `renewsAt`.
- **Billing history is synthesised, not stored.** Kept inline to avoid a `BillingRecord` entity for v1. If real billing records are needed later, add `BillingRecord` (FK → Subscription) and replace `mockBillingHistory`.
- **Single tier.** `premium_monthly` at $9.99 — matches #16's single-plan decision. Annual/lifetime would add tiers to `SubscriptionTier`.
